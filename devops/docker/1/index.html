Docker基础

1. 基本概念

1.1. Docker

1.1.1. 简介

  • 理念：一次构建，处处运行。
  • 三要素：镜像，容器，仓库。
    镜像：只读模板，用于创建Docker容器。
    容器：镜像的一个运行实例，一个精简的Linux环境。
    仓库：集中存放镜像文件，分为公开仓库和私有仓库。
  • CS架构。
    docker守护进程运行在主机上，通过socket连接从客户端访问，守护进程从客户端接收命令并管理运行在主机上的容器。

1.1.2. 作用

  • 更快速的应用交付和部署。
  • 更便捷的升级和扩缩容。
  • 更简单的系统运维。
  • 更高效的计算资源利用。

1.2. 容器化VS传统虚拟化

Docker 虚拟机
操作系统 与宿主机共享OS操作系统。 宿主机上运行虚拟机的OS。
存储大小 镜像小，便于存储和传输。 镜像庞大。
运行性能 几乎没有性能损失。 操作系统消耗额外的CPU和内存。
移植性 轻便灵活适应于linux。 笨重，与虚拟化技术耦合性高。
硬件亲和性 面向软件开发者。 面向硬件维护者。
  • 传统的虚拟机技术是虚拟整套的硬件，在其上运行完整的操作系统，在系统上运行需要的应用进程。
  • 容器则是直接运行在宿主的内核上，容器没有自己的内核，也没有进行硬件虚拟，容器之间相互隔离，每个容器有自己的文件系统，进程之间不会相互影响，能区分计算资源。
    docker比虚拟机快的原因：
    1.docker有着比虚拟机更少的抽象层。
    2.docker利用的是宿主机的内核，而不需要加载操作系统OS内核。

3.3. 容器命令

3.3.1. 新建并启动容器

  • 语法：docker run [options] 镜像名 [command] [args]
  • 选项：
    • –name=”名称”：指定容器名。
    • -d：启动守护式容器，并返回容器ID。
    • -i：以交互模式运行容器，配合-t一起使用。
    • -t：为容器重新分配一个伪输入终端，配合-i一起使用。
    • -P：随机端口映射。
    • -p：指定端口映射。
    • -v：添加数据卷。
    • –privileged=true：扩大容器的权限，使得容器中的root拥有真正的root权限，否则容器内root只是外部的一个普通用户权限。
    • –restart：指定重启策略。no默认策略，在容器退出时不重启容器；always在容器退出时总是重启容器；on-failure在容器非正常退出时重启容器，on-faulure:3标识最多重启三次；unless-stopped在容器退出时重启容器，但是不考虑在docker守护进程启动时就已经停止的容器。
    • –network：指定网桥模式，bridge、host、container:容器ID或者容器名、none、自定义网络。详见Docker网络。
  • 示例：

3.3.2. 查看容器

  • 语法：docker ps [options]
  • 选项：
    • -a：显示所有正在运行和历史运行的容器。
    • -l：显示最近创建的容器。
    • -n：显示最近创建的n个容器。
    • -q：静默模式，只显示容器ID。
    • –no-trunc：不截断输出。
  • 示例：

4. 镜像

4.1. 简介

  • 轻量级可执行的独立软件包，用于打包软件运行环境和基于运行环境开发的软件。
  • union文件系统是docker镜像的基础。docker采用这种分层结构，可以共享资源，例如多个镜像从相同的base镜像构建而来，则在宿主机中只需要保存一份base镜像即可。docker镜像最底层是bootfs。
    Union联合文件系统是分层、轻量级、高性能的文件系统，支持对文件系统的修改作为一次提交来一层层的叠加，同时可以将不同的目录挂载到同一个虚拟文件系统下。特性是一次同时加载多个文件系统，但外面只能看到一个文件系统。

4.2. 特点

  • docker镜像都是只读的。
  • 容器启动时，一个新的可写层被加载到镜像的顶部。这一层叫成容器层，容器层下都是镜像层。

4.3. 提交镜像

语法：docker commit -m=”描述信息” -a=”作者” 容器名或者容器ID 要创建的目标镜像名:[标签名]。

6.1. 作用

  • 容器的持久化。
  • 容器间继承，共享数据。

6.2. 添加数据卷

6.2.1. 命令添加

6.2.2. Dockerfile添加

  2. 构建镜像。
  3. 运行容器。
    容器运行后，便会有默认的数据卷挂载。

6.3. 数据卷容器

  • 容器挂载数据卷，其他容器通过挂载这个父容器实现数据共享，挂载数据卷的容器被称为数据卷容器。
    容器之间配置信息的传递，数据卷的生命周期一直持续到没有容器使用它为止。父容器停止，子容器仍然可以同步主机的数据修改，因为子容器继承的是父容器的挂载规则。
  • 语法：docker run -it –name 名称 –volumes-from 父容器 镜像。

8. DockerFile解析

8.1. 简介

Dockerfile是构建docker镜像的构建文件，是由一系列命令和参数构成的脚本。
每个保留字指令都是大写，且至少一个参数。
从上至下，顺序执行。
#表示注释。
每个指令都会创建一个新的镜像层，并对镜像进行提交。

8.2. Dockerfile执行流程

  1. 从基础镜像运行一个容器。
  2. 执行一条指令并对容器进行修改。
  3. 提交一个新的镜像层。
  4. 基于刚提交的镜像运行一个新的容器。
  5. 执行下一个指令，直到所有的指令都执行完成。

8.3. 构建过程

  1. 编写Dockerfile文件。
  2. 构建镜像。
  3. 运行镜像实例。

8.4. Dockerfile保留字

  • FROM：设置基础镜像。
    FROM java:8
  • MAINTAINER：设置镜像维护者的邮箱和姓名。
    MAINTAINER fengbiao<2014695858@qq.com>
  • RUN：构建需要执行的命令，两种格式shell格式和exec格式。
    RUN ./test.php dev offline # shell格式
    RUN ["./test.php", "dev", "offline"] # exec格式
  • EXPOSE：当前容器对外暴露的端口。
  • WORKDIR：容器工作目录。
  • USER：该镜像以什么样的用户去执行，默认root。
  • ENV：设置环境变量。
    ENV MY_PATH /usr/mytest
  • ADD：将宿主机目录文件拷贝到镜像中，如果压缩包并自动解压。
  • COPY：类似ADD拷贝文件到镜像中，不会解压。
  • VOLUME：容器数据卷，因为不确定因素，此时不需要宿主机目录，待运行时，会自动映射宿主机。
  • CMD：指定容器启动时，运行的命令，可以有多个CMD，但只有最后一个生效，且会被docker run后的参数覆盖。
    CMD 命令 # shell格式
    CMD ["可执行文件", "参数1", ...] # exec格式
  • ENTRYPOINT：指定容器启动时，运行的命令，会被docker run后的参数追加而非覆盖，但是可以采用–entrypoint覆盖。
  • ONBUILD：被继承时触发的命令。
    ONBUILD RUN echo "文本" # 镜像被继承时，便会输出文本

8.5. 虚悬镜像

  • 仓库名和标签都是<none>的镜像。
    'docker build .'等异常情况下产生。
  • 常用命令：
    docker image ls -f dangling=true # 查看虚悬镜像
    docker image prune # 删除虚悬镜像
