    1. 单例模式

    1.1. 作用

    减少性能开销。

    1.2. 应用场景

    • 需要频繁创建销毁的对象。
    • 创建时耗时过多或者耗费资源过多但又经常使用的对象、工具类。
    • 频繁访问数据库或文件的对象（数据源、SessionFactory）、数据库连接池。
    • 读取配置文件的类。
    • 操作系统的文件系统、回收站、任务管理器。
    • 计数器、日志应用。
    • Spring 中 Application、Bean、Servlet、Controller。

    1.3. 代码实现

    1.3.1. 饿汉式

    线程安全、调用效率高、不能延时加载（类初始化时，立即加载）。

    public class Singleton {
    private static Singleton instance = new Singleton();
    private Singleton() {
    }
    public static Singleton getInstance() {
        return instance;
    }
}

    1.3.2. 懒汉式

    线程安全、调用效率不高、可以延时加载。

    public class Singleton {
    private static Singleton instance;
    private Singleton() {
    }
    public static synchronized Singleton getInstance() {
        if (instance == null) {
            instance = new Singleton();
        }
        return instance;
    }
}

    1.3.3. 双重检查

    由于 JVM 底层内部模型原因，可能出现问题，需要加volatile修饰符，避免指令重排序。

    public class Singleton {
    private static volatile Singleton instance; // 防止指令重排
    private Singleton() {
    }
    public static Singleton getInstance() {
        if (instance == null) {
            synchronized (Singleton.class) {
                if (instance == null) {
                    instance = new Singleton();
                }
            }
        }
        return instance;
    }
}

    1.3.4. 静态内部类

    线程安全、调用效率高、可以延时加载。

    public class Singleton {
    private Singleton() {
    }
    public static Singleton getInstance() {
        return Inner.instance;
    }
    public static class Inner { // 内部类只有被调用时才会被加载
        public static Singleton instance = new Singleton();
    }
}

    1.3.5. 枚举类

    线程安全、调用效率高、不能延时加载，不可以通过反射和反序列化来破解。

    public enum Singleton {
    INSTANCE;
    public void hello() {
        System.out.println("hello~");
    }
}

    1.4. 破解单例模式

    1.4.1. 反射

    @Test
public void test() throws Exception {
    Constructor<Singleton> constructor = Singleton.class.getDeclaredConstructor(null);
    constructor.setAccessible(true); // 设置权限
    Singleton singleton = constructor.newInstance(null);
    /**
     * 解决方案：在单例模式类的私有构造方法中，判断实例是否为null，如果非null，则抛出异常。
     */
}

    反射无法破解枚举类的原因：

    Constructor.java
    public final class Constructor<T> extends Executable {
    public final class Constructor<T> extends Executable {
        if ((clazz.getModifiers() & Modifier.ENUM) != 0)
            throw new IllegalArgumentException("Cannot reflectively create enum objects");
    }
}

    1.4.2. 反序列化

    @Test
public void test() throws Exception {
    ByteArrayOutputStream baos = new ByteArrayOutputStream();
    ObjectOutputStream oos = new ObjectOutputStream(baos);
    oos.writeObject(instance); // 要求单例类实现序列化接口
    ByteArrayInputStream bais = new ByteArrayInputStream(baos.toByteArray());
    ObjectInputStream ois = new ObjectInputStream(bais);
    Singleton singleton = (Singleton) ois.readObject();
    /**
     * 解决方案：在单例模式类中，增加反序列化回调方法private Object readResolve() throws ObjectStreamException，返回实例对象。
     */
}

    反序列化无法破解单例的原因：序列化时，仅仅是将枚举类的 name 输出到结果中，反序列化时根据 name 调用 valueOf 查找枚举对象。

    Runtime.java
    public class Runtime {
    private static Runtime currentRuntime = new Runtime();
    public static Runtime getRuntime() {
        return currentRuntime;
    }
    private Runtime() {}
}

    2. 工厂模式

    2.1. 作用

    实现调用者和创建者的分离。

    2.2. 应用场景

    • JDK 中 Calendar 的 getInstance 方法。
    • JDBC 中 Connection 对象的获取。
    • Hibernate 中 SessionFactory 获取 Session。
    • Spring 中 IOC 容器创建和管理 Bean。
    • XML 解析时，DocumentBuilderFactory 创建解析对象。
    • 反射中 Class 对象的 newInstance。

    2.3. 代码实现

    2.3.1. 简单工厂

    不满足开闭原则 ocp，实际情况常采用。

    Shape.java
    public interface Shape {
}
    Circle.java
    public class Circle implements Shape {
}
    Square.java
    public class Square implements Shape {
}
    ShapeSimpleFactory.java
    public class ShapeSimpleFactory {
    // 每次增加Shape类型，都需要修改此类，不满足开闭原则。
    public static Shape getShape(String type) {
        if ("circle".equals(type)) {
            return new Circle();
        } else if ("square".equals(type)) {
            return new Square();
        } else {
            return null;
        }
    }
}

    2.3.2. 工厂方法

    满足开闭原则，但是结构复杂。

    ShapeFactory.java
    public interface ShapeFactory {
    Shape getShape();
}
    CircleShapeFactory.java
    // 每次增加Shape类型，都增加一个类，满足开闭原则。
public class CircleShapeFactory implements ShapeFactory {
    @Override
    public Shape getShape() {
        return new Circle();
    }
}
    SquareShapeFactory.java
    public class SquareShapeFactory implements ShapeFactory {
    @Override
    public Shape getShape() {
        return new Square();
    }
}

    2.3.3. 抽象工厂

    不能增加单个产品，可以增加产品族。

    AbstractFactory.java
    // 抽象工厂
public abstract class AbstractFactory {
    public abstract Shape getShape(String type);
    public abstract Color getColor(String type);
}
    ShapeFactory.java
    // 工厂实现
public class ShapeFactory extends AbstractFactory {
    @Override
    public Shape getShape(String type) {
        if ("circle".equals(type)) {
            return new Circle();
        } else if ("square".equals(type)) {
            return new Square();
        } else {
            return null;
        }
    }
    @Override
    public Color getColor(String type) {
        return null;
    }
}
    FactoryProducer.java
    // 工厂创建者
public class FactoryProducer {
    public static AbstractFactory getFactory(String type) {
        if ("shape".equals(type)) {
            return new ShapeFactory();
        } else if ("color".equals(type)) {
            return new ColorFactory();
        } else {
            return null;
        }
    }
}

    2.4. 源码解析之 Calendar

    Calendar.java
    public abstract class Calendar implements Serializable, Cloneable, Comparable<Calendar> {
    public static Calendar getInstance()
    {
        return createCalendar(TimeZone.getDefault(), Locale.getDefault(Locale.Category.FORMAT));
    }
    private static Calendar createCalendar(TimeZone zone,
                                           Locale aLocale)
    {
        if (aLocale.hasExtensions()) {
            String caltype = aLocale.getUnicodeLocaleType("ca");
            if (caltype != null) {
                switch (caltype) {
                case "buddhist":
                cal = new BuddhistCalendar(zone, aLocale);
                    break;
                case "japanese":
                    cal = new JapaneseImperialCalendar(zone, aLocale);
                    break;
                case "gregory":
                    cal = new GregorianCalendar(zone, aLocale);
                    break;
                }
            }
        }
        if (cal == null) {
            if (aLocale.getLanguage() == "th" && aLocale.getCountry() == "TH") {
                cal = new BuddhistCalendar(zone, aLocale);
            } else if (aLocale.getVariant() == "JP" && aLocale.getLanguage() == "ja"
                       && aLocale.getCountry() == "JP") {
                cal = new JapaneseImperialCalendar(zone, aLocale);
            } else {
                cal = new GregorianCalendar(zone, aLocale);
            }
        }
        return cal;
    }
}

    3. 原型模式

    3.1. 作用

    简化对象创建过程，提高效率。
    浅克隆：对基本类型属性进行值传递，对引用类型属性进行引用传递，仅将引用值（内存地址）复制给新的对象。
    深克隆：对整个对象（包括对象的引用类型）进行拷贝。

    3.2. 应用场景

    new 一个对象比较复杂或者耗时。

    3.3. 代码实现

    3.3.1. 通过实现 Cloneable 接口

    Sheep.java
    @Getter
@Setter
@ToString
public class Sheep implements Cloneable, Serializable {
    private static final long serialVersionUID = 1L;
    private String name;
    private Human user;
    @Override
    protected Object clone() throws CloneNotSupportedException {
        // return super.clone(); // 浅克隆，引用对象是同一个
        Sheep clone = (Sheep) super.clone();
        Human user = (Human) this.user.clone();
        clone.setUser(user);
        return clone;
    }
}

    3.3.2. 通过序列化、反序列化

    Main.java
    @Test
public void test() throws Exception {
    ByteArrayOutputStream baos = null;
    ObjectOutputStream oos = null;
    ByteArrayInputStream bais = null;
    ObjectInputStream ois = null;
    try {
        baos = new ByteArrayOutputStream();
        oos = new ObjectOutputStream(baos);
        oos.writeObject(sheep1); // 需要类以及子类都实现序列化接口，否则会抛出异常
        bais = new ByteArrayInputStream(baos.toByteArray());
        ois = new ObjectInputStream(bais);
        Sheep sheep2 = (Sheep) ois.readObject();
    } catch (IOException e) {
        e.printStackTrace();
    } catch (ClassNotFoundException e) {
        e.printStackTrace();
    } finally {
        if (ois != null) {
            try {
                ois.close();
            } catch (IOException e) {
                e.printStackTrace();
            }
        }
        if (bais != null) {
            try {
                bais.close();
            } catch (IOException e) {
                e.printStackTrace();
            }
        }
        if (oos != null) {
            try {
                oos.close();
            } catch (IOException e) {
                e.printStackTrace();
            }
        }
        if (baos != null) {
            try {
                baos.close();
            } catch (IOException e) {
                e.printStackTrace();
            }
        }
    }
}

    4. 建造者模式

    4.1. 作用

    • 构建模式，一步一步创建一个复杂的对象。将复杂对象的建造过程抽象出来，不同的实现方法构造不同表现形式的对象。
    • 当需要构建一个比较复杂的对象，实现构造 Builder 和装配 Director 的解耦。

    4.2. 应用场景

    • StringBuilder 类中 append 方法。
    • JDBC 中 PreparedStatement。
    • JDOM 中，DomBuilder 和 SAXBuilder。

    4.3. 代码实现

    HouseBuilder.java
    // 抽象建造者
public abstract class HouseBuilder {
    protected House house = new House();
    public abstract void buildBaise();
    public abstract void buildWall();
    public abstract void buildRoffed();
    public House buildHouse() {
        return house;
    }
}
    CommonHouseBuilder.java
    // 具体建造者
public class CommonHouseBuilder extends HouseBuilder {
    @Override
    public void buildBaise() {
        house.setBaise("10m地基");
    }
    @Override
    public void buildWall() {
        house.setBaise("10m高墙");
    }
    @Override
    public void buildRoffed() {
        house.setRoffed("房顶");
    }
}
    CommonHouseBuilder.java
    // 指挥者
public class HouseDirector {
    private HouseBuilder houseBuilder;
    public HouseDirector(HouseBuilder houseBuilder) {
        super();
        this.houseBuilder = houseBuilder;
    }
    public void setHouseBuilder(HouseBuilder houseBuilder) {
        this.houseBuilder = houseBuilder;
    }
    public House buildHouse() {
        houseBuilder.buildBaise();
        houseBuilder.buildWall();
        houseBuilder.buildRoffed();
        return houseBuilder.buildHouse();
    }
}

    4.4. 源码解析之 StringBuilder

    Appendable：抽象建造者。
    AbstractStringBuilder：具体建造者。
    StringBuilder：既充当具体建造者又充当指挥者。

    5. 总结

    创建型设计模式：关注对象的创建。
