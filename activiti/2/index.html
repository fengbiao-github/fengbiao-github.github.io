    3. 个人任务

    3.1. 分配任务负责人

    3.1.1. 固定分配

    业务流程建模时指定固定的任务负责人。例如指定代理人或候选人。

    3.1.2. 表达式分配

    • UEL-Value 表达式分配。
      指定属性名。${属性名}。
      指定对象属性名。${对象名.属性名}。
    • UEL-Method 方法表达式。
      ${user.getName()}
    • UEL-Method 和 UEL-Value 组合使用。
      ${userService.getLeaderName(user.name)}
    • 其他。
      ${leave.days > 3 && leave.days < 7}

    3.1.3. 任务监听器设置

    CustomTaskListener.java
    public class CustomTaskListener implements TaskListener {
    /**
     * Create：任务创建后触发。
     * Assignment: 任务分配后触发。
     * Delete: 任务完成后触发。
     * All: 所有事件发生都触发。
     */
    @Override
    public void notify(DelegateTask delegateTask) {
        System.out.println("事件名称: " +  delegateTask.getEventName());
        System.out.println("节点名称: " +  delegateTask.getName());
        // 可以设置任务负责人。
    }
}

    4. 流程变量

    4.1. 简介

    为流程执行需要而创建，必须实现序列化接口。

    4.2. 作用域

    4.2.1. Global 变量

    一个流程变量作用域为流程实例时，称为 Global 变量。流程变量默认作用域是流程实例。变量名不能重复，设置相同名称的变量，后设置的值会覆盖前设置的变量值。

    4.2.2. Local 变量

    任务和执行实例仅仅是针对一个任务和一个执行实例范围，范围没有流程实例大，称为 Local 变量。Local 变量由于在不同的任务或不同的执行实例中，作用域互不影响，变量名可以相同没有影响。Local 变量名也可以和 Global 变量名相同，没有影响。

    4.3. 设置流程变量

    4.3.1. 设置 Global 变量

    • 流程启动时设置变量。
    @Test
public void test() throws Exception {
    Map<String, Object> params = new HashMap<>();
    UserDO userDO = new UserDO();
    userDO.setName("fengb");
    params.put("user", userDO);
    ProcessInstance processInstance = runtimeService.startProcessInstanceByKey("myLeave", params);
}
    • 任务办理时设置变量。其他节点只有在该任务完成后才可使用。
    @Test
public void test() throws Exception {
    Map<String, Object> params = new HashMap<>();
    UserDO userDO = new UserDO(); // 必须实现序列化接口
    userDO.setName("fengb");
    params.put("user", userDO);
    String taskId = "b041b4d7-b9fb-11ee-8cda-226fd9e449bb";
    taskService.complete(taskId, params);
}
    • 通过当前流程实例设置。流程实例必须未执行完成。
    @Test
public void test() throws Exception {
    Map<String, Object> params = new HashMap<>();
    UserDO userDO = new UserDO(); // 必须实现序列化接口
    userDO.setName("fengb");
    params.put("user", userDO);
    String executionId = "0ed15113-b9b2-11ee-b732-226fd9e449bb";
    // 设置单个变量
    // runtimeService.setVariable(executionId, "user", userDO);
    // 设置多个变量
    runtimeService.setVariables(executionId, params);
}
    • 通过当前任务设置。任务必须未结束，否则报错。
    @Test
public void test() throws Exception {
    Map<String, Object> params = new HashMap<>();
    UserDO userDO = new UserDO(); // 必须实现序列化接口
    userDO.setName("fengb");
    params.put("user", userDO);
    String taskId = "b041b4d7-b9fb-11ee-8cda-226fd9e449bb";
    // 设置单个变量
    // taskService.setVariable(taskId, "user", userDO);
    // 设置多个变量
    taskService.setVariables(taskId, params);
}

    6. 网关

    6.1. 排他网关

    用来在流程中实现决策。排他网关只会选择一个为 true 的分支执行，如果两个分支条件都为 true，排他网关会选择 id 值较小的一条分支去执行。如果从网关出去的线所有条件都不满足则系统抛出异常。
    在连线的condition条件上设置分支条件也可实现分支。缺点：当条件都不满足，流程就结束了（异常结束）。

    6.2. 并行网关

    并行网关在业务应用中常用于会签任务，会签任务即多个参与者共同办理的任务。与排他网关的主要区别是，并行网关不会解析条件，即使定义了条件也会被忽略。

    6.3. 包含网关

    排他网关和并行网关的结合体。和排他网关一样，可以在连线上定义条件表达式，定义的表达式会被包含网关解析。但是包含网关主要区别是也可以不指定表达式，同时选择多条分支，这和并行网关是一样的。

    6.5. 总结

    排他网关：解析条件，流向条件为 true 的节点或者报错（没有满足条件的）。
    并行网关：不解析条件，分支节点都执行。
    包含网关：有条件就解析条件，条件成立就流向，不成立就不流向，全都不成立报错；没有条件直接流向。
