Java8新特性

1、简介

  • 速度更快。

    HashMap底层从Hash表变成Hash表加红黑树。
    ConcurrentHashMap底层采用CAS比较并交换。
    垃圾回收，永久区从堆中分离，为元空间（物理内存）。

  • 代码更少。

    Lambda表达式。

  • 强大的StreamAPI。
  • 便于并行。
  • 最大化减少空指针异常。

*2、Lambda表达式

2.1、语法

需要函数式接口（接口中只有一个抽象方法，即函数式接口）支持。

  • 无参数，无返回值。() -> {…}。
  • 有参数，无返回值。(x, y) -> {…}。
  • 如果只有一个参数，小括号可以省略。x -> {…}。
  • 如果函数体只有一条语句，大括号和return都可以省略。x -> x。
  • 参数类型可以省略，可以通过上下文自己识别。

2.2、示例

3、函数式接口

3.1、简介

接口中只有一个抽象方法，即函数式接口。可以采用注解@FunctionalInterface修饰，用于检查。

3.2、JDK内置函数式接口

  • Consumer<T>: void accept(T t)，消费型接口。
  • Supplier<T>: T get()，供给型接口。
  • Function<T, R>: R apply(T t)，函数型接口。
  • Predicte<T>: Boolean test(T t)，断言型接口。
  • BiFunction<T, U, R>：R apply(T t, U u)。
  • UnaryOperator<T>：T apply(T t)。
  • BinaryOperator<T>：T apply(T t1, T t2)。
  • BiConsumer<T, U>：void accept(T t, U u)。
  • ToDoubleFunction<T>：double applyAsDouble(T t)，同理有ToIntFunction、ToLongFunction。
  • IntFunction<R>：R apply(int value)，同理有DoubleFunction、LongFunction。

4、方法引用与构造器引用

4.1、方法引用

  • 类::静态方法。调用方法的参数列表与返回值类型与函数式接口的函数列表与返回值类型保持一致，可以省略。
  • 对象::实例方法。调用方法发参数列表与返回值类型与函数式接口的函数列表与返回值类型保持一致，可以省略。
  • 类::实例方法。第一个参数是实例方法的调用者，第二个参数是实例方法发参数时，可以省略。

4.2、构造器引用

类::new。

4.3、数组引用

Type[]::new。

*5、Stream流

5.1、简介

流：数据渠道，用于操作数据源（集合、数组等）所生成的元素序列。
流不会存储元素；流不会改变源对象；流操作是延迟执行的，需要结果时，才执行。
流操作步骤：创建流；中间操作；终止操作。

5.2、创建流

  • Collection系列集合提供的stream()和parallelStream()方法。
  • Arrays中静态方法stream()获取数组流。
  • Stream类静态方法of()。
  • 创建无限流。

5.3、中间操作

  • filter：过滤。
  • limit：截断。
  • skip：跳过。
  • distinct：去重。
  • map：将元素转换成其他形式或者提取信息，函数会应用于每一个元素，然后映射成一个新的元素。
  • faltMap：将流中每一个值都转换成另一个流，然后把所有流拼接成另一个流。
  • sorted：排序。

5.4、终止操作

  • 查找与匹配。
    • allMatch：是否匹配所有元素。
    • anyMatch：是否至少匹配一个元素。
    • noneMatch：是否没有匹配所有元素。
    • findFirst：返回第一个元素。返回为Optional容器对象。
    • findAny：返回任意一个，如果串行流，依旧是第一个，一直不变？
    • count：总数。
    • max：返回流中最大的值。
    • min：返回流中最小的值。
  • 规约reduce。
  • 收集。
    • 转换为集合。
    • 计算。
    • 分组。
    • 拼接。

5.5、并行流

5.5.1、定义

将一个内容分成多个数据块，并用不同的线程分别处理每个数据块的流。

5.5.2、Fork/Join框架

在必要的情况将一个大任务，进行拆分成若干个小任务（不可拆分），再将一个小任务运算的结果进行join汇总。

Fork/Join框架与传统线程池的区别：Fork/Join框架采用工作窃取模式。当执行新任务时，会将其拆分成更小的任务执行，并将小任务加到线程队列中，然后再从一个随机线程队列中偷一个将其放在自己的队列。

7、新时间日期

7.1、背景

Date构造时间日期对象，需要计算（年从1990年开始计算，星期从日开始）。
时区TimeZone非线程安全。
时间格式化SimpleDateFormat非线程安全。

7.2、SimpleDateFormat非线程安全

测试SimpleDateFormat非线程安全：

解决方案1：使用ThreadLocal存储SimpleDateFormat。

解决方案2：使用新的日期时间API。

7.3、新时间日期特性

  • 格式化日期时间。
  • LocalDateTime、LocalDate、LocalTime。
  • Instant时间戳，以Unix元年（1970-1-1）到某时刻的毫秒值。
  • Duration时间间隔、Period日期间隔。
  • TemporalAdjuster时间校正器。
  • 时区。

8、注解新特性

8.1、重复注解

jdk8之前不允许重复注解，jdk8以后支持重复注解。

8.2、类型注解

在相应注解上添加作用类型ElementType.TYPE_PARAMETER，目前只能配合框架一起使用，例如@NotNull等注解用来检查变量非空。

8.3、示例

9、Optional容器

9.1、简介

Optional<T>类(java.util.Optional)是一个容器类，代表一个值存在或不存在，原来用null表示一个值不存在，现在Optional可以更好的表达这个概念。并且可以避免空指针异常。

9.2、常用API

  • Optional.of(T t)：创建一个实例。
  • Optional.empty()：创建一个空实例。
  • Optional.ofNullable(T t)：如果t不为null，创建一个实例，否则创建一个空实例。
  • isPresent()：判断实例是否有值。
  • get()：获取实例值。
  • orElse(T t)：如果调用对象包含值，则返回值，否则返回t。
  • orElseGet(Supplier s)：如果调用对象包含值，则返回值，否则返回s获取的值。
  • map(Function f)：如果有值对其处理，并返回处理后的Optional，否则返回一个空实例。
  • flatMap(Function f)：与map类似，但是要求返回值必须为Optional类型。
