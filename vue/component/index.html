    1. 基本概念

    1.1. 组件

    实现应用中局部功能代码和资源的集合。

    1.2. 作用

    复用编码，简化项目编码，提高运行效率。

    1.3. 非单文件组件

    • 模板编写没有提示。
    • 没有构建过程，无法将 ES6 转换成 ES5。
    • 不支持组件的 CSS。
    • 真正开发中几乎不用。

    1.4. VueComponent

    • 组件本质是一个 VueComponent 的构造函数，由 Vue.extend 生成。

      export default Vue.extend({
  name: 'school',
  // ...
})
// 简写
export default {
  name: 'school',
  // ...
}

    • Vue 解析时会创建组件的实例对象，即执行 new VueComponent(options)。

    • 每次调用 Vue.extend 返回的都是一个全新的 VueComponent。

    • 组件配置中，data 函数、methods 中函数、watch 中的函数、computed 中的函数，this 均是 VueComponent 对象；new Vue()配置中，this 均是 Vue 实例对象。

    • VueComponent 的实例对象，简称 vc；Vue 的实例对象，简称 vm。
      vm是可复用的Vue实例，所以与Vue接收相同的选项，仅有像el这样根实例特有的选项。
      一个组件的data选项必须是一个函数，因此每个实例可以维护一份被返回对象的独立拷贝。

    1.5. 重要的内置关系

    VueComponent.prototype.__proto__===Vue.prototype

    这一个重要的内置关系让组件实例对象 vc 可以访问到 Vue 原型上的属性、方法。

    1.6. 单文件组件

    1. 创建组件。

      <template>
  <!-- 页面模板 -->
</template>
<script>
  export default {
    name: 'school',
    data() {
      return {}
    }
  }
</script>
<style>
  /* 样式 */
</style>

    2. 使用组件。

      <template>
  <School></School>
</template>
<script>
  import School from './School'
  export default {
    name: 'App',
    components: { School }
  }
</script>

    2. vue-cli 脚手架

    2.1. 初始化项目

    1. 全局安装@vue/cli。

      npm install -g @vue/cli

    2. 创建项目。

      vue create xxx

    3. 启动项目。

      cd xxx
npm run serve

    2.2. render 函数

    • vue.js 是完整版的 vue，包含核心功能+模板解析器；vue.runtime.xxx.js 是运行版的 vue，只包含核心功能，没有模板解析器。

    • 由于 vue.runtime.xxx.js 没有版本解析器，所以不能是使用 template 配置项，需要使用 render 函数接收到的 createElement 函数去指定具体内容。

      new Vue({
  el: '#app',
  render(createElement) {
    return createElement('h1', '你好')
  };
  // render: (createElement) => createElement('h1', '你好');
  // render: h => h('h1', '你好');
  // render: h => h(APP)
})

    2.3. 配置代理

    module.exports = {
  devServer: {
    proxy: {
      // 匹配请求前缀
      '/api1': {
        target: 'ip:port', // 代理目标的基础路径
        changeOrigin: true, // 如果true，服务器收到的请求头中host是代理目标；如果为false，服务器收到的请求头中host是前端地址
        pathRewrite: { '^/api1': '' } // 去掉请求前缀
      },
      '/api2': {
        target: 'ip:port',
        changeOrigin: true,
        pathRewrite: { '^/api2': '' }
      }
    }
  }
}

    3. ref 属性

    • 用来给元素或子组件注册引用信息（id 的替代品）。

    • 应用在 html 标签上获取的是真实的 DOM 元素，应用在组件标签上是组件实例对象 vc。

      <!-- 打标识 -->
<h1 ref="xxx"></h1>
<School ref="xxx"></School>
<script>
  // 获取
  // this.$refs.xxx
</script>

    4. props 配置项

    • 让组件接收外部传过来的数据。

      <!-- 传递数据 -->
<Student name="AAA" :age="18"></Student>
      export default {
  name: 'Student',
  // 方式1
  props: ['name', 'age'],
  // 方式2
  props: {
    name: String,
    age: number
  },
  // 方式3
  props: {
    name: {
      type: String, // 类型
      required: true // 必须
    },
    age: {
      type: number,
      default: 18 // 默认值
    }
  }
}

    • props 是只读的，vue 底层会监测对 props 的修改，如果进行了修改，就会发出警告，如需修改，可以复制 props 的内容到 data 中，修改 data 中数据。

      export default {
  name: 'Student',
  props: ['name', 'age'],
  data() {
    return {
      myAge: this.age
    }
  },
  methods: {
    addAge() {
      this.myAge++
    }
  }
}

    5. mixin 混入

    5.1. 作用

    可以把多个组件共用的配置提取成一个混入对象。

    5.2. 局部混入

    1. 定义混入。

      mixin.js
      export const mixin1 = {
  data() {
    return {
      x: 100,
      y: 200
    }
  },
  methods: {
    // ...
  },
  mounted() {}
}
export const mixin2 = {
  data() {
    // ...
  },
  methods: {
    // ...
  },
  mounted() {}
}

    2. 使用混入。

      import { mixin1, mixin2 } from './mixin'
export default {
  name: 'Student',
  mixin: [mixin1, mixin2]
}

    5.3. 全局混入

    1. 定义混入。

    2. 使用混入。

      import { mixin1, mixin2 } from './mixin'
Vue.mixin(mixin1)
Vue.mixin(mixin2)

    6. 插件

    6.1. 作用

    用于增强 Vue。

    6.2. 代码实现

    1. 定义插件，一个包含 install 方法的对象。

      export default {
  // install的第一个参数是Vue，第二个以后的参数是插件使用者传递的数据。
  install(Vue, options) {
    // 添加全局过滤器
    // Vue.filter(...)

    // 添加全局指令
    // Vue.directive(....)

    // 配置全局混入
    // Vue.mixin(...,)

    // 添加实例方法
    Vue.prototype.$m1 = () => {}
    Vue.prototype.$p1 = xxx
  }
}

    2. 使用插件。

      import plugins from ./plugins
Vue.use(plugins, 1, 2, 3)

    8. 组件自定义事件

    8.1. 作用

    • 一种组件间通信的方式，适用于：子组件 ===> 父组件。
    • 使用场景： A 是父组件，B 是子组件，B 给 A 传数据，就需要在 A 中给 B 绑定自定义事件，事件的回调在 A 中。

    8.2. 代码实现

    1. 绑定自定义事件。
    <!-- 方式1 -->
<Student @e1="m1"></Student>
<Student v-on:e1="m1"></Student>

<!-- 方式2 -->
<Student ref="stu"></Student>
<script>
  export default {
    data(){
      return {
        stuName: ""
      }
    }
    methods: {
      m1(name) {
        this.stuName = name
      }
    },
    mounted() {
      this.$refs.stu.$on('e1', this.m1)
      this.$refs.stu.$on('e1', name => {
        this.stuName = name
      })
      // this.$refs.stu.$on('e1', function (name) {
      //   // 这里this是触发事件的vc！！！因此绑定自定义事件时，要么配置在methods中，要么使用箭头函数。
      //   this.stuName = name
      // })
    }
  }
</script>
    1. 使用自定义事件。
    // 触发自定义事件
this.$emit('e1', '111')

// 解绑一个自定义事件
this.$off('e1')
// 解绑多个自定义事件
this.$off(['e1', 'e2'])
// 解绑全部的自定义事件
this.$off()

    9. 全局事件总线

    9.1. 作用

    一种组件间通信的方式，适用于任意组件间通信。

    9.2. 代码实现

    1. 安装全局事件总线。
    // 方式1. 不建议
const Demo = Vue.extends({})
Vue.prototype.x = new Demo()

// 方式2
new Vue({
  el: '#app',
  render: h => h(APP),
  beforeCreate() {
    Vue.prototype.$bus = this // 安装全局事件总线，$bus就是当前应用的vm
  }
})
    1. 使用事件总线。
    export default {
  methods: {
    demo(data) {}
  },
  mounted() {
    // 绑定
    this.$bus.$on('xxx', this.demo)
  },
  beforeDestroy() {
    // 解绑当前组件所用到的事件
    this.$bus.$off('xxx')
  }
}
    // 触发
this.$bus.$emit('xxx', xxx)

    10. 消息订阅与发布 pubsub

    10.1. 作用

    一种组件间通信的方式，适用于任意组件间通信。

    10.2. 代码实现

    1. 安装pubsub-js。

      npm i pubsub-js

    2. 订阅消息。

      import pubsub from 'pubsub-js'
export default {
  methods: {
    demo(msgName, data) {}
  },
  mounted() {
    this.pid = pubsub.subscribe('xxx', this.demo) // 订阅消息
  },
  beforeDestroy() {
    pubsub.unsubscribe(this.pid)
  }
}

    3. 发布消息。

      pubsub.publish('xxx', xxx)

    11. nextTick

    在下一次 DOM 更新结束后执行其指定的回调。

    handleEdit(){
  this.edit = true;
  // 因为此时input框还没有出现所以聚焦不会生效
  // this.$refs.inputTitle.focus()
  this.$nextTick(function(){
    this.$refs.inputTitle.focus()
  })
}

    12. 过渡与动画

    12.1. 作用

    在插入、更新或移除 DOM 元素时，在合适的时候给元素添加样式类名。

    12.2. 代码实现

    1. 准备样式。

      <style>
  /** 
  .v-enter-active {
    animation: a1 0.5s linear;
  }
  .v-leave-active {
    animation: a1 0.5s linear reverse;
  }
  @keyframes a1 {
    from {
      transform: translateX(-100%);
    }
    to {
      transform: translateX(0px);
    }
  }
  */
  /** 进入的起点， 离开的终点 */
  .hello-enter,
  .hello-leave-to {
    transform: translateX(-100%);
  }
  /** 进入过程中、离开过程中 */
  .hello-enter-active,
  .hello-leave-active {
    transform: 0.5s linear;
  }
  /** 进入的终点、离开的起点 */
  .hello-enter-to,
  .hello-leave {
    transform: translateX(0);
  }
</style>

    2. 使用。

      <button @click="isShow = !isShow">显示或隐藏</button>
<!-- appear开始的时候就有动画 -->
<transition appear>
  <h1 v-show="isShow">动画与过渡</h1>
</transition>
<!-- 如果指定了name，则样式也需要调整为.name-enter-active -->
<transition name="demo" appear>
  <h1 v-show="isShow">动画与过渡</h1>
</transition>
<!-- 有多个元素需要过渡， 需要使用transition-group，且每个元素都要指定key值 -->
<transition-group name="demo" appear>
  <h1 v-show="isShow" key="1">动画与过渡</h1>
  <h1 v-show="!isShow" key="1">动画与过渡</h1>
</transition-group>

    12.3. 集成第三方动画

    1. 安装 animate。

      npm install animate.css --save

    2. 引入样式。

      import 'animate.css'

    3. 使用。

      <button @click="isShow = !isShow">显示或隐藏</button>
<transition appear name="animate_animated animate_bounce" entre-active-class="animate_swing" leave-active-class="animate_backOutUp">
  <h1 v-show="isShow">动画与过渡</h1>
</transition>

    13. 插槽

    13.1. 作用

    让父组件可以向子组件指定位置插入 html 结构，也是一种组件间通信的方式，适用于父组件 ===> 子组件。

    13.2. 默认插槽

    <template>
  <div>
    <!-- 公共部分 -->
    <!-- 定义插槽 -->
    <slot>
      <!-- 插槽默认内容 -->
    </slot>
  </div>
</template>
    <Category>
  <div>
    <!-- 要插入html结构 -->
  </div>
</Category>

    13.3. 具名插槽

    <template>
  <div>
    <!-- 公共部分 -->
    <!-- 定义插槽 -->
    <slot name="slot1">
      <!-- 插槽默认内容 -->
    </slot>
    <slot name="slot2">
      <!-- 插槽默认内容 -->
    </slot>
  </div>
</template>
    <Category>
  <div slot="slot1">
    <!-- 要插入html结构 -->
  </div>
  <!-- 如果使用div会多一层结构。-->
  <template v-slot:slot2>
    <!-- 要插入html结构 -->
  </template>
</Category>

    13.4. 作用域插槽

    <template>
  <div>
    <!-- 公共部分 -->
    <!-- 定义插槽 -->
    <slot :games="games">
      <!-- 插槽默认内容 -->
    </slot>
  </div>
</template>
    <Category>
  <!-- data可以任意值 -->
  <template scope="data">
    <!-- 要插入html结构 -->
    {{ data.games}}
  </template>
</Category>
<Category>
  <template scope="{games}">
    <!-- 要插入html结构 -->
    {{ games}}
  </template>
</Category>
