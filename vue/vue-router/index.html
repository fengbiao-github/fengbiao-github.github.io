    1. 基本概念

    1.1. vue-router

    vue 的一个插件库，专门用来实现 SPA 应用。

    1.2. SPA 应用

    • 单页 WEB 应用（single page web application，SPA）。
    • 整个应用只有一个完整的页面（index.html），点击页面中导航链接不会刷新页面，只会做页面的局部更新。
    • 数据需要通过 ajax 请求获取。

    2. 基本使用

    1. 安装 vue-router。

      # 查看版本号。
npm view vue-router versions
# vue-router版本需和vue版本对应起来，vue2.x版本使用vue-router3.x版本，vue3.x使用vue-router4.x版本。
npm i vue-router@3.5.6 -S

    2. 定义路由组件。
      路由组件通常存放在 views 文件夹，一般组件通常存放在 components 文件夹。

      views/About.vue
      <template>
  <div>About页面</div>
</template>
<script>
  export default {
    name: 'About',
    mounted() {
      console.log('About组件被挂载', this.$route, this.$router)
    },
    beforeDestroy() {
      console.log('About组件即将被销毁')
    }
  }
</script>
      views/Home.vue
      <template>
  <div>Home页面</div>
</template>
<script>
  export default {
    name: 'Home',
    mounted() {
      console.log('Home组件被挂载', this.$route, this.$router)

      console.log('Home组件即将被销毁')
    }
  }

  router // router: router

    5. 注意事项

      • 通过切换，路由组件默认是被销毁掉的，需要的时候再去挂载。
      • 每个组件都有自己的$route 属性，里面存储着自己的路由信息。
      • 整个应用只有一个 router，可以通过组件的$router 属性获取到。

    4.1. query 参数

    router/index.js
    import VueRouter from 'vue-router'

import About from '@/views/About.vue'
import Home from '@/views/Home.vue'

import MessageDetail from '@/components/MessageDetail.vue'
import Messages from '@/components/Messages.vue'
import News from '@/components/News.vue'

export default new VueRouter({
  routes: [
    {
      path: '/about',
      component: About
    },
    {
      path: '/home',
      component: Home,
      children: [
        {
          path: 'news',
          component: News
        },
        {
          path: 'messages',
          component: Messages,
          children: [
            {
              name: 'msgDetail', // 路由名称
              path: 'detail',
              component: MessageDetail
            }
          ]
        }
      ]
    }
  ]
})
    components/Messages.vue
    <template>
  <div>
    <ul>
      <li v-for="msg in msgList" :key="msg.id">
        <!-- 方式1 -->
        <router-link :to="`/home/messages/detail?id=${msg.id}&title=${msg.title}`">{{ msg.title }}</router-link>
        <!-- 方式2 -->
        <router-link
          :to="{
            path: '/home/messages/detail',
            query: {
              id: msg.id,
              title: msg.title
            }
          }"
          >{{ msg.title }}</router-link
        >
        <!-- 方式3 -->
        <router-link
          :to="{
            name: 'msgDetail',
            query: {
              id: msg.id,
              title: msg.title
            }
          }"
          >{{ msg.title }}</router-link
        >
      </li>
    </ul>
    <router-view></router-view>
  </div>
</template>
<script>
  export default {
    name: 'Messages',

    5. router-link 的 replace 属性

    • 作用：控制路由跳转时操作浏览器历史记录的模式。
    • 浏览器的历史记录有两种写入方式：push 和 replace。push 是追加历史记录，replace 是替换当前记录，路由跳转默认为 push。
    • 开启 replace 模式：<router-link replace active-class="active" to="/home/news">News</router-link>。

    6. 编程式路由导航

    编程式路由导航可能会报"Avoided redundant navigation to current location: 'xxx'错，原因是不允许从当前路由跳转到当前路由。 

    // push跳转
this.$router.push({
  name: 'msgDetail',
  params: {
    id: msg.id,
    title: msg.title
  }
})
// replace跳转
this.$router.replace({
  name: 'msgDetail',
  params: {
    id: msg.id,
    title: msg.title
  }
})
// 后退
this.$router.back()
// 前进
this.$router.forward()
// 跳转。正数前进，负数后退。
this.$router.go(this.n)

    7. 缓存路由组件

    作用：让不展示的路由组件保持挂载不被销毁。

    <!-- 如果不指定include属性则全部生效，否则仅对应组件名生效。 -->
<keep-alive include="News">
  <router-view></router-view>
</keep-alive>

deactivated() {
},
// 路由组件被激活时触发

}

    9. 路由守卫

    9.1. 作用

    对路由进行权限控制。分为全局守卫，独享守卫，组件内守卫。

    9.2. 全局路由守卫

    router/index.js
    import VueRouter from 'vue-router'

import About from '@/views/About.vue'
import Home from '@/views/Home.vue'

import MessageDetail from '@/components/MessageDetail.vue'
import Messages from '@/components/Messages.vue'
import News from '@/components/News.vue'

const router = new VueRouter({
  routes: [
    {
      path: '/about',
      component: About,
      meta: {
        title: '关于'
      }
    },
    {
      path: '/home',
      component: Home,
      meta: {
        title: '首页'
      },
      children: [
        {
          path: 'news',
          component: News,
          meta: {
            title: '新闻'
          }
        },
        {
          path: 'messages',
          component: Messages,
          meta: {
            title: '消息'
          },
          children: [
            {
              name: 'msgDetail',
              path: 'detail/:id/:title', // 占位符
              component: MessageDetail,
              props($route) {
                return {
                  id: $route.params.id, // $route.query.id,
                  title: $route.params.title
                }
              },
              meta: {
                title: '消息详情'
              }
            }
          ]
        }
      ]
    }
  ]
})

// 全局前置路由守卫，初始化时被调用，每次路由切换前被调用
router.beforeEach((to, from, next) => {
  console.log('-->', to, from, next)
  next()
})

// 全局后置路由守卫
router.afterEach((to, from) => {
  document.title = to.meta.title || 'Vue学习项目'
})

export default router

    10. 工作模式

    10.1. hash 模式

    对于url，#及其后面的内容就是hash值，hash值不会包含在HTTP请求中，即hash值不会带给服务器。

    • 地址中带着#号，不美观。
    • 若以后将地址通过第三方手机 app 分享，若 app 校验严格，则地址将被标记为不合法。
    • 兼容性较好。

    10.2. history 模式

    • 地址干净，美观。

    • 兼容性和 hash 模式相比略差。

    • 应用部署上线时需要后端人员支持，解决刷新页面服务端 404 问题。
