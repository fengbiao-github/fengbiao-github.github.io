    1. 基本概念

    1.1. 简介

    一套用于构建用户界面的渐进式 JavaScript 框架。

    1.2. 特点

    • 采用组件化模式，提高代码复用率且让代码更好维护。
    • 声明式编码，让编码人员无需直接操作 DOM，提高开发效率。
    • 使用虚拟 DOM+优秀的 Diff 算法，尽量复用 DOM 节点。

    1.3. HelloWorld

    可以安装Live Server插件进行浏览器访问。

    <!DOCTYPE html>
<html>
  <head>
    <meta charset="utf-8" />
    <title>HelloWorld</title>
  </head>
  <body>
    <div id="root">
      <h1>hello, {{name}}</h1>
    </div>
    <div class="root">
      <h1>hello, {{name}}</h1>
    </div>
  </body>
</html>
<script src="../js/vue.js"></script>
<script type="text/javascript">
  // el写法1
  new Vue({
    // el: document.getElementById('root'),
    el: '#root',

    // data写法1：对象式
    // data: {
    //     name: "vue"
    // }

    // data写法2：函数式，建议！
    data: function () {
      return {
        name: 'vue'
      }
    }
    // data(){
    //     return {
    //         name: "vue"
    //     }
    // }
  })
  new Vue({
    el: '.root',
    data: {
      name: 'vue'
    }
  })
  // el写法2
  const vm = new Vue({})
  vm.$mount('#root')
</script>

    2. 模板语法

    2.1. 插值语法

    用于解析标签体内容。

    <h1>{{name}}</h1>

    2.2. 指令语法

    用于解析标签（标签属性、标签体内容、绑定事件等）。

    <a v-bind:href="url">链接</a>
<!-- 简写 -->
<a :href="url">链接</a>

    3. 数据绑定

    3.1. 单向数据绑定

    数据只能从 data 流向页面。

    <a v-bind:href="url">链接</a> <a :href="url">链接</a>

    3.2. 双向数据绑定

    数据不仅能从 data 流向页面，还能从页面流向 data，v-model 只能应用在表单类元素（input、select）上。

    <input v-model:value="name" />
<!-- 简写 -->
<input v-model="name" />

    4. MVVM 模型

    • M：模型 Model，对应 data 中数据。
    • V：视图 View，模板。
    • VM：视图模型 ViewModel，Vue 实例对象。

    5. 数据代理

    5.1. Object.defineProperty

    let person = {
  name: '张三',
  sex: '男'
}
Object.defineProperty(person, 'age', {
  value: 18,
  enumerable: true, // 控制属性是否可以枚举，默认值是false
  writable: true, // 控制属性是否可以被修改，默认值是false
  configurable: true // 控制属性是否可以被删除，默认值是false
})
let number = 18
Object.defineProperty(person, 'age', {
  get: function () {
    return number
  },
  set: function (value) {
    number = value
  }
})

    5.2. Vue 数据代理

    • 通过 vm 对象来代理 data 对象中属性的操作（读/写），更加方便的操作 data 中的数据。

    • 基本原理：通过 Object.defineProperty()把 data 对象中所有属性添加到 vm 上，为每一个添加到 vm 上的属性都指定一个 getter/setter，在 getter/setter 内部去操作（读/写）data 中对应的属性。

      let obj = { x: 100 }
let obj2 = { y: 200 }
Object.defineProperty(obj2, 'x', {
  get() {
    return obj.x
  },
  set(value) {
    obj.x = value
  }
})

    6. 事件

    6.1. 基本使用

    methods中配置的函数不要使用箭头函数！否则this就不是vm了。

    <html>
  <head>
    <meta charset="utf-8" />
    <title>事件</title>
  </head>

  <body>
    <div id="root">
      <button v-on:click="show1">show1</button>
      <!-- 简写 -->
      <button @click="show1">show1</button>
      <button @click="show2($event, 111)">show2</button>
      <button @click="show3">show2</button>
    </div>
  </body>
  <script src="../js/vue.js"></script>
  <script>
    new Vue({
      el: '#root',
      methods: {
        show1(e) {
          console.log('show1', e)
        },
        show2(e, n) {
          console.log('show2', e)
          console.log('show2', n)
          console.log('show2', this)
        },
        // methods中配置的函数不要使用箭头函数！否则this就不是vm了
        show3: () => {
          console.log('show3', this)
        }
      }
    })
  </script>
</html>

    6.2. 事件修饰符

    • prevent：阻止默认事件。

      <a href="http://baidu.com" @click.prevent="show1">111</a>

    • stop：阻止事件冒泡。

      <div style="background-color: gold;" @click="show2($event, 111)">
  <button style="margin: 50px;" @click.stop="show2($event, 222)">222</button>
</div>

    • once：事件只触发一次。

      <button @click.once="show2($event, 333)">333</button>

    • capture：使用事件的捕获模式。

      <!-- 先捕获，再冒泡，回调事件默认在冒泡阶段执行。不加修饰符点击按钮，先输出555，再输出444。通过添加修饰符点击按钮，会先输出444，再输出555。 -->
<div style="background-color: gold;" @click.capture="show2($event, 444)">
  <button style="margin: 50px;" @click="show2($event, 555)">555</button>
</div>

    • self：只有 event.target 是当前操作的元素是才触发事件。

      <div style="background-color: gold;" @click.self="show2($event, 666)">
  <button style="margin: 50px;" @click="show2($event, 777)">777</button>
</div>

    • passive：事件的默认行为立即执行，无需等待事件的回调执行完毕。

      <html>
  <head>
    <meta charset="utf-8" />
    <title>事件</title>
    <style>
      li {
        height: 100px;
      }
    </style>
  </head>

  <body>
    <div id="root">
      <!-- wheel鼠标滚轮事件，scroll滚动条滚动事件 -->
      <div style="height: 200px;background-color: gray;overflow: auto;" @wheel.passive="gun">
        <ul>
          <li>1</li>
          <li>2</li>
          <li>3</li>
          <li>4</li>
        </ul>
      </div>
    </div>
  </body>
  <script src="../js/vue.js"></script>
  <script>
    new Vue({
      el: '#root',
      methods: {
        gun() {
          for (var i = 0; i < 100000; i++) {
            console.log('@')
          }
          console.log('-----')
        }
      }
    })
  </script>
</html>

    6.3. 键盘事件

    • vue 中常用的按键别名。
      回车 => enter
      删除 => delete（捕获删除和退格键）
      退出 => esc
      空格 => space
      换行 => tab（必须配合keydown使用）
      上 => up
      下 => down
      左 => left
      右 => right

    • vue 未提供别名的按键，可以使用按键原始的 key 值去绑定，但是要转为 kebab-case（短横线命名）。

    • 系统修饰符（ctrl，alt，shift，meta）用法特殊。
      配合keyup使用：按下修饰键的同时按下其他键，随后释放其他键，事件才被触发。
      配合keydown使用：正常触发事件。

    • 可以使用 keyCode 去指定具体的按键，不推荐。

    • 可以定制按键别名。

      <html>
  <head>
    <meta charset="utf-8" />
    <title>hello</title>
  </head>

  <body>
    <div id="root">
      <!-- 按钮别名 -->
      <input @keyup.enter="demo" />
      <!-- 使用keyCode指定具体的按键 -->
      <input @keyup.13="demo" />
      <input @keyup.ctrl="demo" />
      <!-- ctrl+x才触发 -->
      <input @keyup.ctrl.x="demo" />
      <input @keyup.hello="demo" />
    </div>
  </body>
  <script src="../js/vue.js"></script>
  <script>
    Vue.config.keyCodes.hello = 13
    new Vue({
      el: '#root',
      methods: {
        demo(e) {
          console.log(e.target.value)
          console.log(e.keyCode, e.key)
        }
      }
    })
  </script>
</html>

    7. 计算属性

    计算属性不存在，通过已有属性计算得来。

    底层借助了Object.defineproperty方法提供的getter和setter。

    <html>
  <head>
    <meta charset="utf-8" />
    <title>计算属性</title>
  </head>

  <body>
    <div id="root">
      姓：<input v-model="n1" /><br />
      名：<input v-model="n2" /><br />
      姓名：{{n3}}
    </div>
  </body>
  <script src="../js/vue.js"></script>
  <script>
    Vue.config.keyCodes.hello = 13
    new Vue({
      el: '#root',
      data: {
        n1: '',
        n2: ''
      },
      methods: {},
      computed: {
        n3: {
          // 初次读取时执行一次；当依赖的数据发生改变时会被再次调用
          get() {
            return this.n1 + '-' + this.n2
          },
          set(value) {
            this.n1 = value.split('-')[0]
            this.n2 = value.split('-')[1]
          }
        }
        // 简写
        // n3() {
        //   return this.n1 + this.n2;
        // }
      }
    })
  </script>
</html>

    8. 监视属性

    被监视的属性变化时，回调函数自动调用。被监视的属性必须存在。

    computed能完成的功能，watch都可以完成。watch能完成的功能，computed不一定能完成，例如watch可以进行异步操作。

    <html>
  <head>
    <meta charset="utf-8" />
    <title>监视属性</title>
  </head>

  <body>
    <div id="root">
      <h1>{{isHot ? "热" : "冷"}}</h1>
      <button @click="isHot=!isHot">切换</button>
      <h1>{{nums}}</h1>
      <button @click="nums.x++">x++</button>
      <button @click="nums={x: 1, y: 1}">全量替换</button>
    </div>
  </body>
  <script src="../js/vue.js"></script>
  <script>
    Vue.config.keyCodes.hello = 13
    var vm = new Vue({
      el: '#root',
      data: {
        isHot: true,
        nums: {
          x: 1,
          y: 1
        }
      },
      methods: {},
      // 方式1
      watch: {
        isHot: {
          // 初始化时调用一下handler
          immediate: true,
          handler(newValue, oldValue) {
            console.log(newValue, oldValue)
          }
        },
        // 简写
        // isHot: (newValue, oldValue) => {
        //   console.log(newValue, oldValue)
        // },
        nums: {
          // 深度监视，监视内部值的改变
          deep: true,
          // 初始化时调用一下handler
          immediate: true,
          handler(newValue, oldValue) {
            console.log(newValue, oldValue)
          }
        }
      }
    })
    // 方式2
    // vm.$watch('isHot', {
    //   // 初始化时调用一下handler
    //   immediate: true,
    //   handler(newValue, oldValue) {
    //     console.log(newValue, oldValue)
    //   }
    // })
    // 简写
    vm.$watch('isHot', (newValue, oldValue) => {
      console.log(newValue, oldValue)
    })
  </script>
</html>

    <html>

    10. 条件渲染

    • v-if：适用于切换频率较低的场景。不展示的 DOM 元素直接被移除。可以和 v-else-if 和 v-else 一起使用，但要求结构不能被打断。

    • v-show：适用于切换频率较高的场景。不展示的 DOM 元素未被移除，仅仅是使用样式隐藏。

      使用v-if时元素可能无法获取到，v-show一定可以获取到。

    11. 列表渲染

    11.1. 基本使用

    <!-- 遍历字符串 -->
<ul>
  <li v-for="(s, index) in str" :key="index">{{s}}</li>
</ul>
<!-- 遍历对象 -->
<ul>
  <li v-for="(v, k) in obj" :key="k">{{v}}</li>
</ul>
<ul>
  <li v-for="(v, k) of obj" :key="index">{{k}}-{{v}}</li>
</ul>
<!-- 遍历数组 -->
<ul>
  <li v-for="(a, index) in arr" :key="index">{{a}}</li>
</ul>
<!-- 遍历指定次数 -->
<ul>
  <li v-for="(v, index) of 5" :key="index">{{index}}-{{v}}</li>
</ul>

    11.2. key 作用

    • key 是虚拟 DOM 对象的标识，当数据发生变化时，Vue 会根据新数据生成新的虚拟 DOM，随后 Vue 进行新虚拟 DOM 与旧虚拟 DOM 的差异比较，比较规则如下：

      • 旧虚拟 DOM 中找到了与新虚拟 DOM 相同的 key：若虚拟 DOM 中内容没变，直接使用之前的虚拟 DOM；若虚拟 DOM 中内容变了，则生成新的真实 DOM，随后替换掉页面中之前的真实 DOM。
      • 旧虚拟 DOM 中未找到了与新虚拟 DOM 相同的 key：创建新的真实 DOM，随后渲染到页面。

    • 开发中最好使用每条数据的唯一标识作为 key。如果不存在对数据的逆序添加、逆序删除等破坏顺序操作，仅用于渲染列表用于展示，使用 index 作为 key 没问题。

      <html>
  <head>
    <meta charset="utf-8" />
    <title>列表渲染</title>
  </head>

  <body>
    <div id="root" :n="n">
      <input v-model="obj" />
      <button v-on:click="add">add</button>
      <ul>
        <li v-for="(obj, index) of arr" :key="k">
          {{obj}}
          <input />
        </li>
      </ul>
    </div>
  </body>
  <script src="../js/vue.js"></script>
  <script>
    new Vue({
      el: '#root',
      data: {
        obj: '',
        arr: ['a1', 'a2', 'a3']
      },
      methods: {
        add() {
          // 逆序增加！！！异常场景
          this.arr.unshift(this.obj)
          this.obj = ''
        }
      }
    })
  </script>
</html>

    12. 收集表单数据

    <html>
  <head>
    <meta charset="utf-8" />
    <title>收集表单数据</title>
  </head>

  <body>
    <form id="root">
      姓名：<input v-model="username" /><br />

    13.1. 局部过滤器

      new Vue({

    }

  filters: {

    },
    mySlice(value) {

    }
  }

<h1>{{now() | timeFormat | mySlice }}</h1>

    14. 指令

    14.1. 内置指令

    • v-bind：单向绑定解析表达式，可简写为:xxx。

      <a v-bind:href="url">链接</a>
<!-- 简写 -->
<a :href="url">链接</a>

    • v-model：双向数据绑定。

      <input v-model:value="n" />

    • v-for：遍历数组、对象、字符串。

      <!-- 字符串 -->
<ul>
  <li v-for="(s, index) in str" :key="index">{{s}}</li>
</ul>
<!-- 对象 -->
<ul>
  <li v-for="(v, index) in obj" :key="index">{{v}}</li>
</ul>
<ul>
  <li v-for="(v, k) of obj" :key="index">{{k}}-{{v}}</li>
</ul>
<!-- 数组 -->
<ul>
  <li v-for="(a, index) in arr" :key="index">{{a}}</li>
</ul>

    • v-on：绑定事件监听，可简写为@。

      <button v-on:click="add">add</button>
<!-- 简写 -->
<button @click="add">add</button>

    • v-if：条件渲染，动态控制节点是否存在。

    • v-else-if：条件渲染，动态控制节点是否存在。

    • v-else：条件渲染，动态控制节点是否存在。

      <div v-if="n < 3">n &lt; 3</div>
<div v-else-if="n < 5">n &lt; 5</div>
<div v-else>n >= 5</div>

    • v-show：条件渲染，动态控制节点是否展示。

      <div v-show="n < 3">n &lt; 3</div>

    • v-text：向指定节点中渲染文本内容。

      <!--
    与插值语法区别：v-text会替换掉节点中所有的内容，{{xxx}}则不会。
-->
<h1 v-text="msg"></h1>
<!-- msg="<span style='color:red'>msg</span>" -->

    • v-html：向指定节点中渲染包含 html 结构的内容。

      <!--
    与插值语法区别：
        1. v-html会替换掉节点中所有的内容，{{xxx}}则不会。
        2. v-html可以识别html结构。
    有安全性问题：容易导致xss攻击。
-->
<h1 v-html="msg"></h1>
<!-- msg="<span style='color:red'>msg</span>" -->

    • v-cloak：使用 css 配合 v-cloak 可以解决网速慢时页面展示出{{xxx}}的问题。

      <!--
    1. 本质是一个特殊的属性，vue实例创建完毕并接管容器后，会删掉v-cloak属性。
    2. 使用css配合v-cloak可以解决网速慢时页面展示出{{xxx}}的问题。
-->
<style>
  [v-cloak] {
    display: none;
  }
</style>
<h1 v-cloak>{{n}}</h1>

    • v-once：所在的节点在初次动态渲染后就视为静态内容了。

      <!--
    1. 所在的节点在初次动态渲染后就视为静态内容了。
    2. 以后数据的改变不会引起v-once所在结构的更新，可以用于优化性能。
-->
<h1 v-once>n的初始值为{{n}}</h1>

    • v-pre：跳过所在节点的编译过程。

      <!-- 
    1. 跳过所在节点的编译过程。
    2. 可利用它跳过没有使用指令语法、没有使用插值语法的节点，加快编译。
-->
<h1 v-pre>n的值为{{n}}</h1>

    14.2. 自定义指令

    14.2.1. 局部指令

    1. 注册局部指令。

      new Vue({
  el: '#root',
  data: {
    n: 1
  },
  directives: {
    // 方式1. 函数式
    big(element, binding) {
      element.innerText = binding.value * 10
    },
    // 方式2. 对象式
    'big-focus': {
      bind: function (element, binding) {
        element.value = binding.value * 100
      },
      inserted: function (element, binding) {
        element.focus()
      },
      update: function (element, binding) {
        element.value = binding.value * 100
        element.focus()
      }
    }
  }
})

    2. 使用指令。

      <h1>n放大10倍的值为<span v-big="n"></span></h1>
<h2>n放大100倍的值为<input type="text" v-big-focus="n" /></h2>

    14.2.2. 全局指令

    1. 注册全局指令。

      // 方式1. 函数式
Vue.directive('big', function (element, binding) {
  element.innerText = binding.value * 10
})
// 方式2. 对象式
Vue.directive('big-focus', {
  // 指令与元素成功绑定时调用
  bind: function (element, binding) {
    element.value = binding.value * 100
  },
  // 指令所在的元素被插入页面时调用
  inserted: function (element, binding) {
    element.focus() // 无法通过函数式实现，需要元素插入页面后执行！！！
  },
  // 指令所在模板结构被重新解析时调用
  update: function (element, binding) {
    element.value = binding.value * 100
    element.focus()
  }
})

    2. 使用指令。

    14.2.3. 注意事项

    • 指令定义时不加 v-，但使用时要加 v-。
    • 指令名如果是多个单词，要使用 kebab-case 命名方式，不要用 camelCase 命名。

    15.4. 销毁流程

    <html>
  <head>
    <meta charset="utf-8" />
    <title>生命周期</title>
  </head>

  <body>
    <div id="root" :x>
      <h1>{{n}}</h1>
      <button @click="add">add</button>

  <script src="../js/vue.js"></script>
  <script>
    new Vue({
      el: '#root',
      // 会替换div块
      // template:
      //   `<div>
      //     <h1>{{n}}</h1>
      //     <button @click="add">add</button>
      //   </div>`,
      data: {
        n: 1
      },
      methods: {
        add() {
          this.n++
        },
        destroyVm() {
          /**
           * 销毁vue实例：
           *  1. 销毁后借助vue开发者工具看不到任何信息。
           *  2. 销毁后自定义事件会失效，但原生dom事件依然有效。
           *  3. 一般不会在beforeDestroy操作数据，因为即使操作数据，也不会再触发更新流程了。
           */
          this.$destroy()
        }
      },
      // vm中data、methods、指令等都处于可用状态，马上要执行销毁过程。一般在此阶段关闭定时器、取消订阅消息、解绑自定义事件等收尾操作
      beforeDestroy() {
        console.log('beforeDestroy', this)
      },
      destroyed() {
        console.log('destroyed', this)
      }
    })
  </script>
</html>
